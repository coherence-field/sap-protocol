# Structural Alignment Protocol (SAP)

The Structural Alignment Protocol (SAP) is a discipline for determining whether a cognitive architecture is structurally compatible with a generative problem. SAP evaluates alignment through invariant detection, layer correctness, architectural coherence, and drift stability.

SAP functions as an executable protocol composed of 18 canonical artifacts organized into Core, Structural, and External layers.

## Site

The full SAP site, including all artifacts, is available at:

**https://coherence-field.github.io/sap-protocol**

## Structure

- **Core Artifacts (1–8)**  
  Foundational primitives, gates, notation, glossary, membrane rules, and commentary.

- **Structural Artifacts (9–13)**  
  Implementation notes, operational checklists, failure modes, drift signatures, and layer identification.

- **External Artifacts (14–18)**  
  Case skeletons, training protocol, audit protocol, meta‑governance notes, and revision protocol.
